WebGL简介

WebGL is a cross-platform, royalty-free web standard for a low-level 3D graphics API based on OpenGL ES, exposed to ECMAScript via the HTML5 Canvas element. WebGL 1.0 exposes the OpenGL ES 2.0 feature set; WebGL 2.0 exposes the OpenGL ES 3.0 API.

OpenGL® ES is a royalty-free, cross-platform API for rendering advanced 2D and 3D graphics on embedded and mobile systems - including consoles, phones, appliances and vehicles. It consists of a well-defined subset of desktop OpenGL suitable for low-power devices, and provides a flexible and powerful interface between software and graphics acceleration hardware.

OpenGL is the industry's most widely used, supported and best documented 2D/3D graphics API making it inexpensive & easy to obtain information on implementing OpenGL in hardware and software.


简而言之，就是WebGL是暴露于js层的OpenGL一套接口。

OpenGL简介

What is OpenGL?

OpenGL is the name for the specification that describes the behavior of a rasterization-based rendering system. It defines the API through which a client application can control this system. The OpenGL rendering system is carefully specified to make hardware implementations allowable.

Hardware vendors, the people who make GPUs, are responsible for writing implementations of the OpenGL rendering system. Their implementations, commonly called "drivers", translate OpenGL API commands into GPU commands. If a particular piece of hardware is unable to implement all of the OpenGL specification via hardware, the hardware vendor must still provide this functionality, typically via a software-based implementation of the features missing from hardware.

What is NOT OpenGL?

The OpenGL API only deals with rendering graphics. OpenGL does not provide functions for animations, timing, file IO, image file format processing, GUI, and so forth. OpenGL is concerned only about rendering.

GLUT is not OpenGL. It is not a part of OpenGL; it is simply a library that is used by some users to create an OpenGL window.

Is OpenGL Open Source?

No, OpenGL doesn't have any source code. GL is a specification which can be found on this website. It describes the interface the programmer uses and expected behavior. OpenGL is an open specification. Anyone can download the spec for free. This is as opposed to ISO standards and specifications, which cost money to access.


简而言之，OpenGL是一套用于图形渲染的跨平台API规范或者讲是协议，规范的实现者是各个硬件厂商。通过这个API，应用程序能够使用GPU的能力。

从API应用场景来讲，OpenGL用于桌面程序的开发，OpenGL ES应用于移动开发，WebGL应用于Web开发。

WebGL的运行机制：WebGL是一个JS API，通过和浏览器进行通信，然后完成渲染过程。通过WebGL的官方介绍，应该是浏览器通过OpenGL ES实现了渲染功能，但其实在Windows上底层使用DX实现，见Chrome WebGL Windows 上实现分析。

  这种技术的诞生方式在软件世界里已经司空见惯的事情了。软件的世界里没有空中楼阁，一切都是协议与抽象：软件的运行最终都要落到操作系统以及CPU这一层，所有编程语言的只是各自定义了一套协议而已。然后有个系统讲"你按照我规定的指令，我给你翻译为一个完整的程序，你自己去在CPU上跑吧”，这就是native的编程语言，系统指编译器，然后另外又有一个系统讲“你按照我规定的指令告诉我需要做什么，然后我帮你做。”，这个被规定的指令同样是编程语言，这个系统于java叫做编译器和虚拟机的东西，于js是浏览器。跨平台就是第二种的类似实现，是一种加中间层的实现方式。

结语

本次之所以由WebGL入手讲解3D渲染过程，其中一个原因是JS运行和开发环境特别容易获取，浏览器足以。另外JS是现在比较流行的语言之一，容易入门。后面通过对渲染流程的学习，相信对于使用OpenGL的进行开发也会大有帮助。

虽然OpenGL在Windows上没有DX流行，行将被Vulkan取代，但是了解其工作的原理对其他类似技术肯定有所裨益，毕竟它们可能在同一种GPU设备运行。虽然我没有实际接触过Vulkan，但是曾经略微使用过DX9，DX9和OpenGL的很多概念是一致的，比如都有一种抗锯齿方式叫做MULTISAMPLE，不一样的可能是实现细节，比如DX9 矩阵行主序、OpenGL矩阵列主序。
